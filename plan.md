# Slopify Implementation Plan

**Domain:** slopify.felixweinberger.com

## Architecture

```
User opens GitHub Issue
        ↓
Felix adds "approved" label
        ↓
GitHub Actions triggers claude-code-action
        ↓
Claude implements feature, creates PR
        ↓
CI runs (lint, build, preview deploy)
        ↓
Auto-merge on CI success
        ↓
Cloudflare Pages deploys to production
```

## Phase 4: In-App Requests, Auth & Storage (Future)

### Goal
Enable users to submit feature requests directly in the app, with authentication and database storage.

### Authentication: GitHub OAuth
- Users log in with GitHub (they already have accounts)
- Pages Functions handle OAuth callback at `/functions/auth/github/callback.ts`
- User's GitHub token used to create issues on their behalf
- Env vars: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`

### Storage: Cloudflare D1 + Drizzle ORM
- D1 (SQLite) is built into Pages, free tier generous
- Drizzle ORM for type-safe schema + auto-generated migrations
- Claude can create schema files, run `drizzle-kit generate`

**Initial schema:**
```typescript
export const users = sqliteTable('users', {
  id: text('id').primaryKey(),           // GitHub user ID
  username: text('username').notNull(),
  avatarUrl: text('avatar_url'),
  accessToken: text('access_token'),      // Encrypted
  createdAt: integer('created_at'),
});
```

### Architecture
```
Frontend (static)
    ↓
Pages Functions (/functions)
    ├── /auth/github/index.ts       - Start OAuth
    ├── /auth/github/callback.ts    - Handle callback, store user
    ├── /auth/logout.ts             - Clear session
    ├── /api/me.ts                  - Get current user
    └── /api/submit-request.ts      - Create GitHub issue
    ↓
D1 Database
    └── users table
```

### Request Flow
1. User logs in with GitHub OAuth
2. User fills "Submit a Request" form in app
3. POST to `/api/submit-request` with title + description
4. Function uses user's GitHub token to create issue in repo
5. Return issue URL to user

### claude-build.yml (Workflow)

```yaml
name: Claude Build Feature

on:
  issues:
    types: [labeled]

jobs:
  build:
    if: github.event.label.name == 'approved'
    runs-on: ubuntu-latest
    permissions:
      contents: write
      issues: write
      pull-requests: write
      id-token: write
    steps:
      - uses: actions/checkout@v4

      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: |
            Implement the feature requested in this issue.
            Follow CLAUDE.md guidelines.
            Create a PR when done.

            ISSUE #${{ github.event.issue.number }}
            TITLE: ${{ github.event.issue.title }}

            ${{ github.event.issue.body }}

      # Auto-merge is enabled, but PR only merges after CI passes
      # Requires branch protection with required status checks
      - name: Enable auto-merge
        run: gh pr merge --auto --squash
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
